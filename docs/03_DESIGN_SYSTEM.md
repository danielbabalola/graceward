# 03 Design System

## Design Direction

Graceward should feel calm, premium, reflective, and spiritually mature.

The app should be warm and illustrated where useful, but not childish.

## Visual Keywords

- Quiet
- Gentle
- Trustworthy
- Prayerful
- Reflective
- Clean
- Premium
- Human
- Peaceful

## Navigation

MVP bottom navigation:

```text
Today | Journal | Prayer | Gratitude | Settings
```

Do not create separate bottom tabs for:

- Faithfulness
- Lessons
- Review

These should live inside other sections.

## UI Principles

1. Prioritize calm over density.
2. Use large readable text.
3. Avoid cluttered dashboards.
4. Use progressive disclosure.
5. Keep daily reflection one tap away.
6. Make privacy settings easy to understand.
7. Show clear loading states during AI processing.
8. Use empty states that encourage without guilt.

## Illustration Style

Use:

- Soft vector illustrations
- Journals
- Lamps
- Light
- Water
- Sunrise
- Paths
- Stones of remembrance
- Calm human scenes

Avoid:

- Cheesy religious clipart
- Excessive crosses everywhere
- AI-looking faces
- Mascot in MVP
- Busy backgrounds

## Motion

Use subtle motion:

- Recording pulse
- Gentle breathing animation
- Processing shimmer
- Smooth page transitions
- Small answered-prayer celebration
- Light haptics

Avoid:

- Overly playful gamification
- Aggressive streak animations
- Guilt-based progress states

## Empty States

Examples:

### Journal Empty

"Start with one honest reflection. You do not need perfect words."

### Prayer Empty

"Add something you want to bring before God."

### Gratitude Empty

"Notice one mercy from today, no matter how small."

### Review Empty

"Reviews become richer as you reflect over time."

## Accessibility

- Support Dynamic Type.
- Maintain strong color contrast.
- All touch targets should be at least 44x44.
- Do not rely on color alone.
- Voice features should have text alternatives.
- Loading states should be readable by screen readers.
