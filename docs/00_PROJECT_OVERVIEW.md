# 00 Project Overview

## Product Name

**Graceward: Daily Reflection**

## Tagline

**Pause. Reflect. Remember God's faithfulness.**

## Product Summary

Graceward is a private Christian reflection app that helps users process their day through voice or text, turning daily thoughts into prayer points, gratitude, biblical reflection, spiritual insights, and reminders of God's faithfulness over time.

It is not simply a journal, a prayer tracker, or an AI devotional app. It is a private spiritual reflection system that helps users bring their real daily life before God.

## Core User Experience

The primary user action is:

> I want to reflect on my day before God.

The user first chooses:

1. **Free Flow**
2. **Guided Reflection**

Then chooses:

1. **Speak**
2. **Type**

Free Flow is for rambling honestly. Guided Reflection is for structured spiritual processing.

## MVP Guided Modes

1. Regular Reflection
2. Lament
3. Rejoice

## Future Guided Modes

1. Conflict
2. Decision
3. Relationship
4. Gratitude
5. Scripture Meditation / Hagah Mode

## Product Identity

Graceward should feel:

- Private
- Peaceful
- Reflective
- Scripture-rooted
- Gentle
- Pastoral
- Emotionally honest
- Spiritually mature
- Secure
- Premium

Graceward should not feel:

- Cheesy
- Gimmicky
- Overly academic
- Generic motivational AI
- Like a public social app
- Like a sermon assistant
- Like a replacement for church, counseling, or pastoral care

## Theological Lane

Graceward is rooted in a high and healthy view of Scripture.

It is not branded narrowly as Reformed, charismatic, denominational, or tied to a public teacher. Its biblical perspective is shaped by:

1. Scripture
2. Responsible hermeneutics
3. Historic Christian orthodoxy
4. JTTE sermons and notes behind the scenes
5. Curated Christian wisdom cards
6. The user's own journal, prayer, gratitude, and faithfulness history

## Local-First Principle

Graceward should be local-first and hybrid-cloud.

Guiding principle:

> Local by default. Cloud when useful. AI only with purpose and permission.

As much as possible should be stored on-device first. Cloud is used for auth, optional sync, subscriptions, AI processing, push notifications, backups, and admin support metadata.

## Core Sections

Recommended MVP bottom navigation:

```text
Today | Journal | Prayer | Gratitude | Settings
```

Deeper features:

- Faithfulness lives inside Gratitude.
- Lessons live inside Journal and Review.
- Review starts as a card/experience launched from Today or Journal rather than a top-level tab.
- Journal should support both timeline and calendar views.

## MVP Success Statement

The MVP succeeds if a user can say:

> I spoke honestly for 10 minutes, and Graceward helped me turn that into prayer, gratitude, biblical reflection, and remembrance of God's faithfulness.
